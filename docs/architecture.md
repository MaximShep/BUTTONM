# Architecture Notes

## Auth

Авторизация сделана для локального MVP: логин и пароль, пользователь создается вручную через seed или Prisma. После входа сервер выдает подписанную HTTP-only cookie.

Отдельной таблицы сессий нет. Все страницы и server actions получают текущего пользователя через `requireUser()`.

## Data Access

Пользователь видит только свои проекты:

- dashboard фильтрует `Project` по `userId`
- страница проекта ищет `Project` по `id` и `userId`
- страница сценария проверяет владельца через связь `project.userId`

## Future Modules

- `lib/llm` - клиенты LLM-провайдеров и логирование
- `lib/prompts` - сборка промптов
- `lib/files` - чтение исходных материалов и будущих загрузок
- `lib/export` - экспорт сценариев
